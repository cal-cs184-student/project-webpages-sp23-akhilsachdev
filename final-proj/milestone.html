Current Progress

As of now we’ve worked on aligning the images so that they can be processed into HDR. Our strategy involves: (1) using a Harris corner detector to find all corners in our pictures, which are used as features, (2) utilizing Adaptive Non-Maximal Suppression to remove half the points in order to reduce the runtime while maximizing the probability of good results, (3) extracting a 40x40 chunk of the images at each point we choose, and compare features in each image to find the best matches, (4) we then use those points and RANSAC to compute a good homography to warp one image to another, and (5) transform one of the images into the space of the other image, which aligns them.

Preliminary Results

We are able to align two images together automatically and without input from humans. The code is super un-optimized though, especially as images get larger.

Progress Reflection

We are somewhat on track compared to our initial goals. We have an unoptimized auto-aligner and a decent plan of how to go about our code.

Future Plans

Based on our current progress, there are a few things that need to be changed before our final product. The first thing that needs to be improved is the runtime; right now the code is a lot slower than it should be. To do this we’re gonna try using numba’s jit and numpy parallelization. Another thing that we need to optimize is to make the program work for more than just pairs of images. We also could want to implement automatic cropping so that it’s easier for us to crop out the unnecessary parts of the image or optimize the code to have more overlap between images.

We also plan on implementing HDR after image alignment is fully implemented and optimized. Our current strategy is to use bracketed exposure. We take our sharpest image to be our reference image. We then divide the reference image into regions of similar light intensities. We then take the best-exposed regions from the burst images and merge them into the corresponding region in the reference image.
